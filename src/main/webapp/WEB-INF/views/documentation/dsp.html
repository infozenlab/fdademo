GSA Requirement

Response

1

assigned one leader, gave that person authority and responsibility, and held that person accountable for the quality of the prototype submitted

Jeff Highman was assigned as a leader by the Company CEO to be accountable for the end-to-end delivery of the solution, process, and delivery of the product within one week. Results of the development were reported to the CEO, the day prior to delivery.

2

assembled a multidisciplinary and collaborative team including a minimum of 5 labor categories from the Development Pool labor categories to design and develop the prototype

Assembled a Core team of 5, including delivery manager, technical architect, front-end developer, DevOps engineer, and Agile coach

3

understand what people need, by including people in the prototype development and design process

Assembled part time secondary stakeholders from within the office that included interns, executives, and other users.

4

used at least three "human-centered design" techniques or tools

Developed Personas to represent consumers of the application, stakeholders that were interested in the process and engineers that were interested in the technology. Determined a business hypothesis around the data set, that an application could help a user identify trends by visualizing an aggregated view of the data set. Developed wireframes and user navigation map

5

created or used a design style guide and/or a pattern library

A style guide was developed to enable responsive controls. We utilized a style guide that was published by a federal agency (USPTO Design Patterns: https://uspto.github.io/designpatterns/). This was extended to support key features such as type-ahead dropdown.

6

performed usability tests with people

Held a focus group to provide feedback on the application. The first iteration was shipped as a minimum viable product on the third day. Conducted mobile device usability tests as well.

7

used an iterative approach, where feedback informed subsequent work or versions of the prototype

Feedback was collected that guided revisions to the layout, navigation, and function. Adjustments were made to the back end to accommodate aggregation constraints. Changes were continuously integrated, deployed on a production system, and tested with a layered approach.

8

created a prototype that works on multiple devices, and presents a responsive design

The responsiveness of the application was tested, and confirmed on mobile devices including tablet and phone. A collapsible navigation bar and a dynamic content pane were used to maximize real estate on each device.

9

used at least five modern and open-source technologies, regardless of architectural layer (frontend, backend, etc.)

Spring (MVC & Security), D3.JS, JQuery, Bootstrap, AspectJ, Jackson, SLF4J, Maven, Jenkins, Vagrant

10

deployed the prototype on an Infrastructure as a Service (IaaS) or Platform as a Service (PaaS) provider, and indicated which provider they used

Deployed on VMware V-Sphere and AWS.

11

wrote unit tests for their code

Wrote unit tests with Junit. Used Mockito to isolate components. Used Selenium to create a UI smoke test.

12

set up or used a continuous integration system to automate the running of tests and continuously deployed their code to their IaaS or PaaS provider

Created a CI pipeline that leveraged Maven, Jenkins, Sonar Cube, and Vagrant for continuous integration. Tests were performed with each check-in. Code quality reports were produced with each build. The tomcat container was automatically built with Vagrant.

13

set up or used configuration management

Configuration items were tracked with Git and Nexus Library manager.

14

set up or used continuous monitoring

Used Ratproxy to scan the application, identified and corrected vulnerabilities.

15

deploy their software in a container (i.e., utilized operating-system-level virtualization)

Deployed to Tomcat 7.

16

make use of a API, by either consuming or providing one RESTfully.

Consumed the openFDA API to aggregate data over successive calls using RESTful client. Processed results with Jackson. Provided a Restful service to enable the aggregation to be consumed.

17

used an iterative approach, where feedback informed subsequent work or versions of the prototype

Conducted daily feedback sessions that were used to refine the form, flow, and function of the application.

provided sufficient documentation to install and run their prototype on another machine

Provided documentation in Readme file to install the application on another machine. Provided instructions for windows and Linux.

18

prototype and underlying platforms used to create and run the prototype are openly licensed and free of charge

The underlying platform was tested on CentOS which is openly licensed. Verified all libraries, including transitive dependencies, were openly licensed. Every technology used in the application, CI pipeline, and DevOps platform was open source.
